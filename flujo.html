Flujo de Trabajo Típico con Git

Este es el proceso estándar que seguirás en la mayoría de tus proyectos:

  1. 1️⃣ Crear o clonar repositorio

    Si es nuevo: git init

    Si existe: git clone URL

  2. 2️⃣ Realizar cambios en los archivos

    Edita el código como necesites en tu editor favorito.

  3. 3️⃣ Verificar el estado

    git status - Revisa qué archivos han cambiado.

  4. 4️⃣ Preparar los cambios

    git add . - Agrega todos los archivos al staging.

    O git add archivo.txt - Agrega archivos específicos.

  5. 5️⃣ Crear commit

    git commit -m "Mensaje descriptivo de los cambios"

  6. 6️⃣ Enviar al remoto

    git push origin main - Sube tus commits a GitHub.

Consejos para buenos commits

  • Usa mensajes claros y descriptivos
  • Un commit por funcionalidad o corrección
  • Evita mensajes como "cambios" o "fix"
  • Usa el presente: "añade función" en lugar de "añadí función"
